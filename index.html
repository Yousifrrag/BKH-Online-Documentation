Module Customization

E-Learning Module

Custom fields and enhancements added to the default Odoo e-Learning module to support BKH's online cycle workflow.

01
Instructor Field
E-Learning

A new Instructor field has been added to the course form under the Options tab → Course section. This field is a Many2one reference to res.partner with a domain filter that restricts the selection to contacts with the type Teacher only.

  • Model: slide.channel
  • Field Type: Many2one → res.partner
  • Domain: Contacts with type = Teacher
  • Location: Options tab → Course section
Options tab showing the new Instructor field (highlighted) under the Course section
02
Total Sessions & Duration (Computed)
E-Learning

Two new computed fields have been added to the course form under the Options tab → Display section:

Field Description Logic
Sessions Count Counts only content lines (excludes sections) Computed from slide.slide content records
Duration Sums the duration of all content lines Auto-populated in the Duration field

These fields automatically calculate as content is added to the course, providing a live summary of the total sessions and total duration.

Display section showing Sessions Count (6) and Duration (01:03) — both auto-computed from content lines
Module Customization

Inventory Module

Enhancements to the product form to support the online cycle's combo products and automated actions.

03
Combo Product — Create on Order Actions
Inventory

Inside the Inventory module, the Product Type: Combo has been enhanced with the ability to trigger automated actions when an order is placed. The Create on Order field now supports two new action types:

  • Course Access — Automatically grants the customer access to an e-Learning course
  • Event Registration — Automatically registers the customer for an event

This allows BKH to sell combo products that bundle course access and event registration together as a single purchasable item.

Product form — Combo type with the Create on Order field (highlighted) supporting course access and event registration actions
Website Implementation

Website Module

The website was implemented using default Odoo Website sections and building blocks. No custom development was made to the website module itself — all sections and implementation were done with standard Odoo tools. Many of the customizations are tailored for the client's mobile application needs.

04
Website Sections & Pages
Website

The following key areas of the website were configured using Odoo's default website builder:

  • Homepage — Hero section with App Store / Play Store download buttons and BKH branding
  • Courses Carousel — A dynamic carousel showcasing available courses with categories, descriptions, and durations
  • Courses Page — Course listing with popularity ranking, levels, session count, and gamification features (ranks & badges)
  • Footer — Social media links (Facebook, Instagram, TikTok, YouTube), contact info, and quick navigation

Note: Some website customizations may appear irrelevant to Odoo's default behavior as they were specifically tailored for the client's mobile application requirements.
